# Customer Journey Map
## Aware
### Trigger Event
- Developer or system administrator encounters a problem with agent reachability or management.
- Research online for solutions or ask colleagues for advice.

### Friction Points
- Lack of visibility into agent status and performance.
- Inefficient manual processes for agent management.
- Limited understanding of agent capabilities and limitations.

### User Emotions
- Frustration
- Confusion
- Annoyance

### Opportunities to Delight
- Provide clear and concise agent status updates.
- Offer automated agent management tools.
- Offer educational resources on agent capabilities and limitations.

### Metric
- Time to resolution (TTR) for agent-related issues.
- Number of manual processes eliminated.

## Consider
### Trigger Event
- Developer or system administrator discovers agent-ops through online research or word of mouth.
- Evaluates agent-ops against existing solutions or manual processes.

### Friction Points
- Lack of trust in new tool or solution.
- Concerns about compatibility with existing systems.
- Limited understanding of agent-ops features and benefits.

### User Emotions
- Skepticism
- Caution
- Curiosity

### Opportunities to Delight
- Showcase agent-ops features and benefits through demos or trials.
- Highlight success stories and customer testimonials.
- Offer free trials or pilot programs to build trust.

### Metric
- Number of trials or pilot programs initiated.
- Net Promoter Score (NPS) for agent-ops.

## Try
### Trigger Event
- Developer or system administrator decides to try agent-ops.
- Begins evaluating agent-ops features and performance.

### Friction Points
- Complexity of agent-ops setup and configuration.
- Limited understanding of agent-ops capabilities and limitations.
- Concerns about data security and privacy.

### User Emotions
- Excitement
- Hope
- Anxiety

### Opportunities to Delight
- Provide clear and concise onboarding and documentation.
- Offer dedicated support and training.
- Highlight data security and privacy measures.

### Metric
- Number of users who complete onboarding and setup.
- User satisfaction ratings for agent-ops support and training.

## Adopt
### Trigger Event
- Developer or system administrator decides to adopt agent-ops.
- Begins using agent-ops as a primary tool for agent management.

### Friction Points
- Integration with existing systems and workflows.
- Limited understanding of agent-ops customization and configuration.
- Concerns about scalability and performance.

### User Emotions
- Confidence
- Satisfaction
- Relief

### Opportunities to Delight
- Provide seamless integration with existing systems and workflows.
- Offer customization and configuration options.
- Highlight scalability and performance capabilities.

### Metric
- Number of users who adopt agent-ops as a primary tool.
- Revenue growth from agent-ops sales.

## Expand
### Trigger Event
- Developer or system administrator decides to expand agent-ops usage.
- Begins exploring additional features and capabilities.

### Friction Points
- Limited understanding of agent-ops advanced features and capabilities.
- Concerns about data integration and analytics.
- Limited understanding of agent-ops customization and configuration.

### User Emotions
- Excitement
- Curiosity
- Confidence

### Opportunities to Delight
- Provide clear and concise documentation for advanced features and capabilities.
- Offer dedicated support and training for data integration and analytics.
- Highlight customization and configuration options.

### Metric
- Number of users who expand agent-ops usage.
- Average revenue per user (ARPU) for agent-ops.
